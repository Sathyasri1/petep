Processor is a simple extension that allows you to define rules for automatic replacement and other modification of PDUs.

Process rule

For each modification you have to create rule:

  • Name: for clarity.
  • Description: for clarity.
  • Enabled
  • Tag:
    • "": Rule will be applied to any PDU.
    • "some_tag": Rule will be applied only to PDUs with specified tag.
  • Rest of the configuration depends on the processor module.
  • Occurrence:
    • -1: All occurrences will be replaced.
    • >= 0: Only specified occurrence (starting from 0), will be replaced (0., 1., 2., ...).
  • What: what should be replaced.
  • With: [What] will be replaced with [With].

Process group

Process groups contain process rules. These groups can be then bound with particular process interceptors.

It is possible to bind one process group to multiple interceptors, so it is up to you, if you will create just one group or multiple groups.

Process interceptor

In order to setup automatic modification, you have to create process interceptors in one or both directions and you have to specify, which process group should these interceptors use.

Internal processors

Replace

Replace is a simple processor that allows you to setup automatic replacement rules.

  • Occurrence:
    • -1: All occurrences will be replaced.
    • >= 0: Only specified occurrence (starting from 0), will be replaced (0., 1., 2., ...).
  • What: what should be replaced.
  • With: [What] will be replaced with [With].
